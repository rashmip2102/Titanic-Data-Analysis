# 🚢 Titanic Data Analysis

A step-by-step data science project analyzing the Titanic passenger dataset — covering data cleaning, exploratory data analysis (EDA), and preprocessing in preparation for machine learning.

---

## 📁 Project Structure

```
├── TheTitanic.csv                # Raw Titanic passenger dataset (1,302 rows × 10 columns)
├── DataCleaningTitanic.ipynb     # Step 1: Data cleaning
├── EDATitanic.ipynb              # Step 2: Exploratory data analysis
└── PreprocessingTitanic.ipynb    # Step 3: Feature engineering & preprocessing
```

---

## 📊 Dataset

The raw dataset (`TheTitanic.csv`) contains **1,302 passenger records** with the following features:

| Column | Description |
|---|---|
| Serial Number | Row identifier |
| Pclass | Passenger class (1 = First, 2 = Second, 3 = Third) |
| Survived | Survival outcome (0 = Perished, 1 = Survived) |
| Name | Passenger name |
| Gender | Passenger gender |
| Age | Passenger age |
| Family | Number of family members aboard |
| Fare | Ticket fare paid |
| Embarked | Port of embarkation (S = Southampton, C = Cherbourg, Q = Queenstown) |
| Date | Date of embarkation |

---

## 📓 Notebooks

### 1. `DataCleaningTitanic.ipynb` — Data Cleaning

Handles the messy raw data to produce a clean, analysis-ready CSV (`TheTitanicCleaned.csv`).

**Steps covered:**
- Remove duplicate rows
- Rename unnamed/incorrectly named columns
- **Age:** Strip non-numeric characters using regex; fill missing values with median age grouped by `Pclass` and `Gender`
- **Gender:** Infer missing values from passenger name titles (Mr., Mrs., Miss, Master)
- **Fare:** Remove stray `*` symbols; fill missing values with group median
- **Embarked:** Normalize whitespace and casing; fill missing values with the mode
- **Date:** Strip `#######` placeholder strings; parse into `datetime`; fill missing with mode date
- Export cleaned dataset as `TheTitanicCleaned.csv`

---

### 2. `EDATitanic.ipynb` — Exploratory Data Analysis

Visualizes patterns and relationships in the cleaned data.

**Analyses included:**

**Univariate**
- Survival count (Survived vs. Perished)
- Age distribution (histogram + KDE)
- Fare distribution (histogram + KDE)

**Bivariate**
- Survival rate by **Gender** — Did women fare better?
- Survival rate by **Passenger Class** — Did wealth affect survival?
- Survival probability vs. **number of family members** onboard

**Multivariate**
- Survival rate by **Class × Gender** combined — e.g., did 3rd-class women survive as well as 1st-class women?
- **Correlation heatmap** of all numeric features

---

### 3. `PreprocessingTitanic.ipynb` — Preprocessing for ML

Transforms the cleaned data into a machine-learning-ready format.

**Steps covered:**
1. **Drop irrelevant columns** — `Serial Number`, `Name`, `Date`
2. **Encode categorical features**
   - `Gender` — One-Hot Encoding (binary)
   - `Embarked` — One-Hot Encoding (3 categories)
3. **Feature scaling** — StandardScaler applied to `Age` and `Fare` to normalize across different ranges
4. **Train/test split** — 80% training / 20% testing using `train_test_split` with `random_state=42`

## 🔍 Key Findings

- **Gender** was a strong predictor of survival — women had significantly higher survival rates
- **Passenger class** correlated with survival: 1st-class passengers fared much better than 3rd-class
- **Family size** had a non-linear effect — small families survived better than solo travelers or very large groups
- **Age** and **Fare** distributions were right-skewed, requiring cleaning and scaling before modeling
